# Heart Disease Risk Detector — Project Documentation

A Flask-based web application developed by Ankan Hazra & Soumyadeep Biswas that predicts heart disease risk using a machine learning classification model trained on the UCI Heart Disease (Kaggle) dataset.

## Setup & Run Instructions

### 1. Install Dependencies

```bash
pip install -r requirements.txt
```

### 2. Download Dataset

Download `heart.csv` from [Kaggle](https://www.kaggle.com/datasets/johnsmith88/heart-disease-dataset) and place it in the `/dataset` folder.

### 3. Create Databases

Open SQLite (or use DB Browser for SQLite) and run the queries listed in the **Database Queries** section below.

### 4. Train the Model

```bash
python train_model.py
```

This will:
- Load and clean the dataset
- Compare Random Forest, Gradient Boosting, and Logistic Regression
- Select the best model based on 5-fold cross-validated AUC score
- Save the model to `models/heart_model.pkl`

### 5. Run the App

```bash
python app.py
```

Visit `http://127.0.0.1:5000` in your browser.

## Database Queries

Run these SQL statements to create both databases before starting the app.

### user.db

```sql
CREATE TABLE users (
    user_id  INTEGER PRIMARY KEY AUTOINCREMENT,
    name     TEXT    NOT NULL,
    phone    TEXT    NOT NULL,
    email    TEXT    NOT NULL UNIQUE,
    username TEXT    NOT NULL UNIQUE,
    password TEXT    NOT NULL         -- stored as bcrypt hash via werkzeug
);
```

### record.db

```sql
CREATE TABLE records (
    record_id  INTEGER PRIMARY KEY AUTOINCREMENT,
    user_id    INTEGER NOT NULL,       -- foreign key → users.user_id
    age        REAL,
    sex        REAL,
    cp         REAL,
    trestbps   REAL,
    chol       REAL,
    fbs        REAL,
    restecg    REAL,
    thalach    REAL,
    exang      REAL,
    oldpeak    REAL,
    slope      REAL,
    ca         REAL,
    thal       REAL,
    prob_yes   REAL,                   -- predicted probability of heart disease (0–1)
    prob_no    REAL,                   -- predicted probability of no heart disease
    risk_label TEXT,                   -- e.g. "Low Risk", "Critical Risk"
    timestamp  TEXT                    -- datetime string: YYYY-MM-DD HH:MM:SS
);
```

> **Note:** SQLite does not enforce foreign keys by default. If you want referential integrity, run `PRAGMA foreign_keys = ON;` at connection time.

## Dataset Features

| Feature    | Description                                           | Type    |
|------------|-------------------------------------------------------|---------|
| `age`      | Patient age in years                                  | Integer |
| `sex`      | 1 = Male, 0 = Female                                  | Binary  |
| `cp`       | Chest pain type (0–3)                                 | Ordinal |
| `trestbps` | Resting blood pressure in mmHg                        | Integer |
| `chol`     | Serum cholesterol in mg/dl                            | Integer |
| `fbs`      | Fasting blood sugar > 120 mg/dl (1 = Yes, 0 = No)    | Binary  |
| `restecg`  | Resting ECG results (0, 1, 2)                         | Ordinal |
| `thalach`  | Maximum heart rate achieved                           | Integer |
| `exang`    | Exercise-induced angina (1 = Yes, 0 = No)             | Binary  |
| `oldpeak`  | ST depression induced by exercise vs. rest            | Float   |
| `slope`    | Slope of peak exercise ST segment (0, 1, 2)           | Ordinal |
| `ca`       | Major vessels coloured by fluoroscopy (0–3)           | Integer |
| `thal`     | Thalassemia: 0=Normal, 1=Fixed defect, 2=Reversible   | Ordinal |
| `target`   | 1 = Heart disease present, 0 = No heart disease       | Binary  |

## Machine Learning Approach

### Algorithms Evaluated

1. **Random Forest** — Ensemble of decision trees; robust and non-linear. Handles feature interactions well. No feature scaling needed.
2. **Gradient Boosting** — Sequential ensemble; each tree corrects errors of the previous. Strong on tabular data.
3. **Logistic Regression** — Linear baseline. Fast, interpretable, uses StandardScaler for normalization.

### Selection Criterion

All three are evaluated via **5-fold stratified cross-validation** with **ROC-AUC** as the scoring metric. The model with the highest mean AUC is selected and saved.

### Risk Thresholds

| Probability of Heart Disease | Risk Level     | Color    |
|------------------------------|----------------|----------|
| 0% – 25%                     | Low Risk       | 🟢 Green  |
| 25% – 50%                    | Moderate Risk  | 🟡 Yellow |
| 50% – 75%                    | High Risk      | 🟠 Orange |
| 75% – 100%                   | Critical Risk  | 🔴 Red    |

## Application Flow

```
User visits /
    ↓
Fills health form (blanks → average values assumed)
    ↓
POST /predict → Model inference → JSON response
    ↓
Results shown: Risk badge + Pie chart + Critical factors
    ↓
[Not logged in] → Prompt to sign up or login at bottom
[Logged in]     → Record saved, past records shown
```

## Security Notes

- Passwords are hashed using **Werkzeug's PBKDF2** (not plain text).
- No OTP / email verification (as specified for this college project scope).
- Change `app.secret_key` to a random string before any public deployment.
